class Options extends AnyRef

This class offers the LogFire options that a user can set. Each option is a variable that (1) has a default value, and (2) can be re-assigned with an assignment statment. All options are spelled with capital letters. The Monitor class extends the Options class. Hence these option variables are accessible on a monitor object as public variables.

  4. var COLOR: Boolean

    When set to true: various messages will be printed out in colors.

    When set to true: various messages will be printed out in colors. If the output medium does not support color printing, color characters will appear anyway, which will be confusing. Therefore the default value is false.

  5. var HISTORY: Boolean

    When set to true: will for each generated fact store the set of events and facts that led to the generation of this fact.

    When set to true: will for each generated fact store the set of events and facts that led to the generation of this fact. This history information will for example be printed as error traces when errors are reported. The flag is by default set to true. It is recommended to set it to false in case time and memory usage becomes a problem.

  6. var PRINT: Boolean

    When set to true: will have the same effect as setting both PRINT_EVENTS and PRINT_FACTS to true.

  7. var PRINT_EVENTS: Boolean

    When set to true: will cause printing of consumed events and user-added facts.

    When set to true: will cause printing of consumed events and user-added facts. Is by default set to false.

  8. var PRINT_FACTS: Boolean

    When set to true: will cause printing of fact memory after each event/fact processed.

    When set to true: will cause printing of fact memory after each event/fact processed. It is by default set to false.

  9. var PRINT_PROGRESS: Boolean

    When set to true: will cause printing of error messages, and other progress information, on standard out.

    When set to true: will cause printing of error messages, and other progress information, on standard out. The flag is by default set to true. It can be set to false in cases, where results are processed in other ways automatically, for example in tests.

  10. var STOP_AFTER_FAIL: Boolean

    When set to true: will cause the monitor to stop after the first error in the trace has been encountered.

    When set to true: will cause the monitor to stop after the first error in the trace has been encountered. If false, the monitor will continue monitoring the trace after a fail. The flag is by default set to false.
